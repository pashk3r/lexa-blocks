# Сценарии использования (Use Cases) — Lexa-Blocks

**Версия:** 3.0 (20.05.2026)

---

## UC-1. Размещение фигуры

### Основной сценарий (Happy Path)

1. Игрок видит 3 фигуры в нижней панели.
2. Игрок зажимает левую кнопку мыши на одной из фигур.
3. Система выделяет фигуру, она следует за курсором.
4. Игрок перемещает фигуру на игровое поле.
5. Система показывает проекцию фигуры на сетке.
6. Игрок отпускает кнопку над свободными ячейками.
7. Система проверяет возможность размещения.
8. Фигура размещается на поле.
9. Система проверяет заполненные линии, очищает их и начисляет очки.
10. Если панель опустела — генерируется новый набор из 3 фигур.
11. Система проверяет условие тупика.

### Альтернативный сценарий A1: размещение в занятую ячейку

1. Игрок отпускает фигуру, и она частично или полностью накладывается на занятые ячейки.
2. Система отклоняет размещение.
3. Фигура возвращается на исходное место в панели выбора.

### Альтернативный сценарий A2: размещение за границей поля

1. Игрок отпускает фигуру так, что она частично выходит за пределы сетки.
2. Система отклоняет размещение.
3. Фигура возвращается на исходное место в панели выбора.

## UC-2. Очистка линий и начисление очков

### Основной сценарий (Happy Path)

1. Игрок успешно разместил фигуру на поле.
2. Система проверяет все строки и столбцы.
3. Если строка полностью заполнена — она очищается.
4. Если столбец полностью заполнен — он очищается.
5. Очки начисляются: 10 очков за каждую очищенную линию.
6. Очки за размещение фигуры плюсуются: по 1 очку за каждый куб в фигуре.
7. Если очищено несколько линий одновременно — очки за линии суммируются.
8. Общий счёт обновляется.
9. Проверяется повышение уровня (каждые 100 очков).
10. Обновляется прогресс-бар.

## UC-3. Наступление тупика

### Основной сценарий (Happy Path)

1. После успешного размещения фигуры система проверяет все 3 фигуры в панели выбора.
2. Ни одну из трёх фигур нельзя разместить на поле.
3. Игра переходит в состояние `LOSS_MENU`.
4. Игрок видит экран Game Over с кнопками «Начать заново» и «Ответить на вопрос».

## UC-4. Спасение через математический вопрос

### Основной сценарий (Happy Path)

1. Игра в состоянии `LOSS_MENU`.
2. Игрок нажимает кнопку «Ответить на вопрос».
3. Система отправляет запрос к внешнему API.
4. API возвращает текст вопроса по высшей математике, четыре варианта ответа (A, B, C, D) и правильный вариант.
5. Система показывает окно с вопросом и вариантами ответа.
6. Игрок выбирает правильный вариант.
7. Система проверяет ответ через API.
8. Ответ правильный.
9. Окно закрывается.
10. Игрок получает 3 спасательных куба.
11. Счёт и поле остаются без изменений.
12. Игра возвращается в состояние `PLAYING`.

### Альтернативный сценарий A1: неправильный ответ

1. (Шаги 1–6 как выше).
2. Игрок выбирает неправильный вариант.
3. Система проверяет ответ через API.
4. Ответ неправильный.
5. Выполняется полный сброс: счёт = 0, уровень = 1, поле чистое, кубы = 0.
6. Игра начинается заново (`PLAYING`).

### Альтернативный сценарий A2: отказ от вопроса

1. В окне вопроса игрок нажимает кнопку «Сдаться».
2. Это засчитывается как неправильный ответ.
3. Выполняется полный сброс.
4. Игра начинается заново.

### Исключение E1: ошибка соединения с API

1. Система не может соединиться с API.
2. Игрок видит сообщение об ошибке и кнопку «Повторить».
3. Игрок нажимает «Повторить» — система пробует снова.
4. Если соединение восстановлено — вопрос загружается.
5. Если нет — игрок может нажать «Сдаться» для перезапуска игры.

## UC-5. Использование спасательного куба

### Основной сценарий (Happy Path)

1. У игрока есть спасательные кубы (больше нуля).
2. Игрок выбирает спасательный куб для размещения.
3. Игрок указывает свободную ячейку на поле.
4. Система размещает куб 1×1.
5. Счётчик кубов уменьшается на 1.
6. Система проверяет заполненные линии, очищает их и начисляет очки.

### Альтернативный сценарий A1: нет кубов

1. У игрока 0 кубов.
2. Выбор куба недоступен.
