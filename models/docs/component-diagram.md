# Текстовое описание диаграммы компонентов – Lexa-Blocks

## Общее назначение
Диаграмма показывает высокоуровневую архитектуру системы Lexa-Blocks, акцентируя разделение на игровое ядро и образовательный модуль, а также взаимодействие с AI-сервисом.

## Компоненты и интерфейсы

### 1. Компонент `GameCore` (игровое ядро)
Отвечает за основную игровую механику: управление полем, фигурами, очками и состояниями.

### 2. Компонент `EduModule` (образовательный модуль)
Реализует генерацию математических вопросов и проверку ответов. Может работать локально или обращаться к внешнему API.

### 3. Интерфейс `IEduService`
Определяет контракт для образовательного сервиса (методы `generateQuestion` и `verifyAnswer`). Этот интерфейс отделяет игровое ядро от конкретной реализации.

## Связи между компонентами

- `GameCore` **зависит от** интерфейса `IEduService`.  
  Это означает, что игровое ядро знает только об интерфейсе и не зависит от конкретной реализации образовательного модуля. Зависимость показана пунктирной стрелкой, что соответствует архетипу "использует" (use).

- `EduModule` **реализует** интерфейс `IEduService`.  
  Образовательный модуль предоставляет конкретную реализацию всех методов, объявленных в интерфейсе. Это отношение реализации (realization).

- `EduModule` **вызывает** облачный сервис `Remote AI API` с пометкой "optionally calls".  
  Это означает, что модуль может (но не обязан) отправлять запросы к внешнему API для генерации вопросов или проверки ответов. Локальная реализация (без вызова API) также возможна.
