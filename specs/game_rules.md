# Игровые правила Lexa-Blocks

## Поле
- Размер 8×8 ячеек.

## Фигуры
- 5 типов: квадрат 2×2, линия 3×1 вертикальная, линия 3×1 горизонтальная, угол L, одиночный куб.
- В панели всегда 3 случайные фигуры. Новый набор появляется только когда все три размещены.

## Очки и прогресс
- За каждый куб размещённой фигуры игрок получает 1 очко.
- За каждую полностью заполненную строку или столбец (8 ячеек) игрок получает 10 очков.
- Если за один ход очищено несколько линий, очки суммируются (например, 2 линии = 20 очков, 3 линии = 30 очков).
- Прогресс = текущий счёт % 100 (0–99). Отображается как процент до следующего уровня.
- Уровень = floor(счёт / 100) + 1. Каждые 100 очков повышают уровень.

## Спасательные кубы
- За правильный ответ на вопрос по Python игрок получает 3 куба размером 1×1.
- Кубы **добавляются** к уже имеющимся в руке (не заменяют их).
- Кубы можно использовать в любой момент после их получения (до окончания игры).
- Лимита на количество кубов нет.

## Условие тупика (Game Over)
- После каждого успешного размещения фигуры проверяются все 3 фигуры в панели.
- Если ни одну из них нельзя разместить, игра переходит в состояние `STATE_GAME_OVER`, затем в `STATE_CHOICE` (экран выбора типа викторины).

## Спасение через вопросы по Python
- Доступно после окончания игры на экране выбора.
- Игроку предлагается **три типа вопросов**:

### 1. Варианты ответа (choice)
- Генерируется вопрос с четырьмя вариантами ответа (A, B, C, D).
- Игрок выбирает один вариант.
- **Правильный ответ** → добавляются 3 спасательных куба, игра продолжается.
- **Неправильный ответ** → полный сброс (счёт=0, поле очищено, кубы=0, уровень=1).

### 2. Ввод ответа (input)
- Генерируется вопрос, ответом на который является одно слово, число или короткое выражение (например, «append», «len», «42»).
- Игрок вводит ответ в текстовое поле (клавиатура, Enter или кнопка «Подтвердить»).
- **Правильный ответ** → добавляются 3 спасательных куба, игра продолжается.
- **Неправильный ответ** → полный сброс.

### 3. Соответствие (match)
- Показываются два столбца: левый (термины, понятия) и правый (определения, описания).
- Игрок должен ввести правильные пары в формате `a1b2c3d4`, где буква — элемент левого столбца, цифра — правого.
- **Правильное соответствие** → добавляются 3 спасательных куба, игра продолжается.
- **Неправильное соответствие** → полный сброс.

### Случайный вопрос
- Кнопка «Случайный вопрос» выбирает один из трёх типов (choice / input / match) случайным образом.

### Общие правила для всех типов
- Вопросы генерируются внешним API (DeepSeek) на тему языка Python (синтаксис, встроенные функции, типы данных, ООП, стандартная библиотека).
- Таймаута на ответ нет.
- При ошибке соединения с API показывается локальный вопрос-заглушка того же типа, позволяющий игроку продолжить (с вариантами «Повторить» или «Сдаться»). Если игрок сдаётся – засчитывается как неправильный ответ (полный сброс).
- При правильном ответе: счёт и поле не меняются, добавляются 3 спасательных куба.
- При неправильном ответе или сдаче: полный сброс (счёт=0, поле очищено, кубы=0, уровень=1).

## Примечание
- Спасательные кубы (одиночные блоки 1×1) после получения размещаются как обычные фигуры. Их можно перетаскивать на любые свободные ячейки. За каждый такой куб начисляется 1 очко, и он может участвовать в очистке линий.
- Если в руке уже есть кубы, новые добавляются к ним (например, было 2 куба, после спасения станет 5).
